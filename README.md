 Library Management System – Project Description

The **Library Management System** is a console-based Java application developed using **Low Level Design (LLD)** principles and **Object-Oriented Programming (OOP)** concepts. The system is designed to simulate the core functionalities of a real-world library, allowing efficient management of books, members, and borrowing activities.

This application enables librarians to add and manage books, register members, track book borrowing and returns, and maintain records of transactions. Each member can borrow a limited number of books, and the system ensures proper validation such as checking book availability and preventing duplicate borrowing. It also maintains borrowing details including the borrow date and due date.

The system automatically calculates fines if a book is returned after the due date. In addition, it provides useful reports such as member borrowing history, book availability status, and a list of top borrowers based on the number of books borrowed.

The project follows a modular design with separate classes for **Book**, **Member**, **BorrowRecord**, and **LibraryService**, ensuring clear separation of responsibilities and better maintainability. Java collections such as **HashMap**, **ArrayList**, and the **Java Time API** are used to efficiently manage and track data within the system.

Overall, the project demonstrates practical implementation of **LLD design patterns, data management, and core Java programming concepts** through a structured and interactive console-based application.
